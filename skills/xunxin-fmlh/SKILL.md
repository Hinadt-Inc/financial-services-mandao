# 迅信-履约指数 · FMLH 1.4.0

## SKILL 定位

本 SKILL 是4个迅信产品中**最精简的快速初筛工具**。履约指数只做一件事：**判断申请人近期是否发生过逾期**。

返回两态结果：

- `result_code=1`：**A(Overdue)**——有逾期记录，触发风险信号
- `result_code=4`：**U**——无数据，无法判断

相比信用探查指数（QJTZ），履约指数不返回正常履约（B-Normal）和延迟履约（B-Delay）状态，也不返回履约正向字段（履约金额/笔数等）。其核心价值在于：**以最低成本完成逾期命中筛查**，适合高并发的贷前批量初筛场景。

**产品选择建议**：

| 场景                                      | 推荐产品                             |
| ----------------------------------------- | ------------------------------------ |
| 只需快速判断"有无逾期"                    | ✅ 履约指数（FMLH）—— 成本低，速度快 |
| 需要区分正常/延迟履约，或需要履约正向字段 | 信用探查指数（QJTZ）                 |
| 需要逾期+共债双维度                       | 全景指数（QJDA）                     |
| 需要建模或多维度精细化分析                | 综合指数V2（ZX-RadarV2）             |

## MCP 依赖

- 必选：`mandao-company`（`getXunxinFmlhV140`）

## 通用执行原则

**第一，履约指数只有两态，不能做出"正常"结论。** `result_code=4`（无数据）≠ 无风险，只代表该用户在三方支付体系内无借贷记录，不可据此直接通过申请。

**第二，`result_code=1` 命中后，核心分析重点是逾期严重程度。** 一旦确认有逾期，应重点看 `max_overdue_days`（逾期天数）和 `currently_overdue`（当前逾期机构数）来评估是历史轻微逾期还是当前严重违约。

**第三，`currently_overdue` 是当前实时状态，优先级高于历史数据。** 若 `currently_overdue ≥ 1`，说明申请人当前仍在逾期中，属于最高风险信号，应直接拒绝。

**第四，`latest_overdue_time` 反映逾期时间衰减。** 距今超过24个月的历史逾期，风险权重显著低于近6个月逾期，可差异化处理（如需降额而非直接拒绝）。

**第五，`acc_exc`（异常还款）是欺诈信号的辅助指标。** 异常还款通常指代扣失败、退款撤销等异常操作，`acc_exc > 0` 需结合其他信号判断是否存在刻意规避还款行为。

## 工作流

### Step 1：调用 MCP 获取数据

```
mcp__mandao-company__getXunxinFmlhV140
参数：{ "idNo": "<身份证号>", "idName": "<姓名>", "phoneNo": "<手机号>" }
```

### Step 2：判断业务返回码

| data.code | 处理方式                                           |
| --------- | -------------------------------------------------- |
| `0`       | 查询命中，进入 Step 3 解读 `result_detail`         |
| `1`       | 未命中，`result_detail=null`，输出"数据未命中"结论 |
| `9`       | 系统异常，提示重新查询，不作风险结论               |

### Step 3：result_code 双态判断

| result_code | 含义                       | 处理方向                        |
| ----------- | -------------------------- | ------------------------------- |
| `1`         | **A(Overdue)**：有逾期记录 | 🔴 进入 Step 4 评估逾期严重程度 |
| `4`         | **U**：无数据/未知         | ⚪ 进入 Step 5 输出未命中结论   |

### Step 4：逾期严重程度评估（result_code=1 时执行）

**当前状态（最高优先级）**：

| 字段                    | 风险解读                                                                 |
| ----------------------- | ------------------------------------------------------------------------ |
| `currently_overdue`     | 当前逾期机构数：`= 0` 历史逾期已解决，`≥ 1` 当前仍在逾期（直接拒绝信号） |
| `currently_performance` | 当前履约机构数：了解申请人当前在贷规模                                   |
| `acc_exc`               | 异常还款机构数：> 0 需关注                                               |

**逾期深度（历史逾期评估）**：

| 字段                  | 风险解读                                                     |
| --------------------- | ------------------------------------------------------------ |
| `max_overdue_days`    | 最长逾期天数区间（见下方分级）                               |
| `max_overdue_amt`     | 最大逾期金额区间                                             |
| `latest_overdue_time` | 最近逾期时间：距今 ≤ 6个月为近期逾期，> 24个月可考虑衰减处理 |

**逾期天数风险分级**：

| `max_overdue_days`                              | 严重程度                     |
| ----------------------------------------------- | ---------------------------- |
| `1-15` 天                                       | 轻微，可结合其他因素降额准入 |
| `16-30` 天                                      | 中等，建议人工复核           |
| `31-60` 天                                      | 较重，建议拒绝或严格限制     |
| `61-90` 天                                      | 严重，建议拒绝               |
| `>90` 天（`91-120`/`121-150`/`151-180`/`>180`） | **极严重，直接拒绝**         |

### Step 5：综合风险定级

| 风险等级          | 触发条件                                                                                      | 准入建议                                     |
| ----------------- | --------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 🔴 **高风险**     | `currently_overdue ≥ 1`（当前仍在逾期），或 `max_overdue_days ≥ 31天`                         | **拒绝**                                     |
| 🟠 **中高风险**   | `result_code=1` + `currently_overdue=0` + `max_overdue_days 16-30天` + 近期逾期               | **人工复核**                                 |
| 🟡 **中风险**     | `result_code=1` + `currently_overdue=0` + `max_overdue_days 1-15天` + 逾期时间较远（>12个月） | **审慎准入，降额处理**                       |
| ⚪ **数据未命中** | `code=1` 或 `result_code=4`                                                                   | 建议配合其他产品综合判断，不单独作为通过依据 |

> 注意：履约指数不返回 `result_code=2`（正常履约）或 `3`（延迟履约），无法主动给出"低风险"正向结论

## 输出模板

- **章节 1：履约状态结论 · Decision Pack**（命中/未命中 + 风险等级 + 准入建议）
- **章节 2：逾期明细分析**（逾期天数/金额/时间/当前逾期机构数，仅 result_code=1 时输出）
- **章节 3：综合评估与处置建议**（基于逾期严重程度的差异化处置方案）
- **章节 4：产品局限说明**（如需正向履约信号，建议升级查询信用探查指数 QJTZ）
- **章节 5：数据来源与查询时间戳**

## 参数

- `--idNo`：身份证号（必填），末位 X 须大写
- `--idName`：姓名（必填）
- `--phoneNo`：手机号（可选，提升命中率）
- `--format md|json`：输出格式，默认 md

## 边界与免责

履约指数仅判断逾期/无数据两态，**不具备正向信用评估能力**。对于 `result_code=4`（无数据）的用户，本产品无法提供任何风险判断依据。

本产品覆盖三方支付数据体系，不包含银行信贷、消费金融持牌机构的正式征信数据，建议配合央行征信报告或其他数据产品综合使用。

逾期天数和金额均以区间形式返回（非精确值），风险评估时应以区间下限作为保守估计基础。
