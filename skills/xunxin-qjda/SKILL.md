# 风控-全景指数 · QJDA 1.3.0

## SKILL 定位

本 SKILL 服务于消费信贷机构贷前风险初筛场景。全景指数通过三方支付数据，同时覆盖**逾期**和**共债**两个维度：

- **逾期维度**：还原申请人近6个月内在各平台的逾期行为，区分超短期（S型）和多期（M型）账期，标注是否已结清
- **共债维度**：追踪近6个自然月的多头借贷重叠情况，识别疑似"借新还旧"的高风险行为模式

单次查询同时获得两维度数据，是贷前初筛的高性价比产品，尤其适合对击中率、拒绝率、风险提升度有量化要求的策略评估场景。

## MCP 依赖

- 必选：`mandao-company`（`getXunxinQjdaV130`）

## 通用执行原则

**第一，先确认入参完整性。** `idNo`（身份证号）和 `idName`（姓名）为必填，`phoneNo` 可选但能提升命中率。身份证末位含 X 时须大写。

**第二，`data.code` 是业务判断的第一入口。** `code=0` 才有 `result_detail`，`code=1` 表示该用户无数据记录（未命中），`code=9` 为系统异常，三种情况处置逻辑完全不同，不可混淆。

**第三，逾期是否结清（`endFlag`）是风险定性的关键分水岭。** `endFlag=N`（未结清）的当前逾期是最强拒绝信号，`endFlag=Y`（已结清）的历史逾期需结合时间远近和金额大小综合判断。

**第四，"借新还旧"是欺诈与流动性危机的复合信号。** `new_or_old=Y` 表明申请人在用新借贷偿还旧债，是信贷机构重点关注的行为模式，触发后须提升风险等级。

**第五，共债机构数反映资金压力。** `current_org_count ≥ 5` 意味着申请人同期在5家以上机构有借贷，高共债本身即是风险因子，需结合逾期情况综合评估。

## 工作流

### Step 1：调用 MCP 获取数据

```
mcp__mandao-company__getXunxinQjdaV130
参数：{ "idNo": "<身份证号>", "idName": "<姓名>", "phoneNo": "<手机号>" }
```

### Step 2：判断业务返回码

| data.code | 处理方式                                                                |
| --------- | ----------------------------------------------------------------------- |
| `0`       | 查询命中，进入 Step 3 解读 `result_detail`                              |
| `1`       | 未命中，无逾期/共债记录，输出"数据未命中"结论，建议结合其他产品综合判断 |
| `9`       | 系统异常，提示重新查询，不作风险结论                                    |

### Step 3：逾期维度解读

从 `result_detail` 提取以下关键指标并逐项评估：

| 字段                                  | 风险解读                               |
| ------------------------------------- | -------------------------------------- |
| `member_count`                        | 逾期机构数，≥3 为高风险信号            |
| `order_count`                         | 逾期订单总数，数量越多风险越高         |
| `debt_amount`                         | 逾期总金额，金额越大风险越高           |
| `debt_detail[].endFlag=N`             | **当前未结清逾期**，触发即为最高风险   |
| `debt_detail[].billType` 含 `M0`/`S0` | 当期逾期（最新账期），风险高于历史逾期 |
| `debt_detail[].endFlag=Y`             | 历史已结清逾期，结合时间评估衰减       |

**账期类型说明**：

- `S` 型（超短期）：7/14/21天账期；`S0`=当期，`S1`=逾期1个账期
- `M` 型（多期）：30天+账期；`M0`=当期，`M1`=逾期1个账期，以此类推
- 数字越大，逾期越严重

### Step 4：共债维度解读

| 字段                              | 风险解读                                 |
| --------------------------------- | ---------------------------------------- |
| `current_org_count`               | 近1月共债机构数，≥5 为高共债             |
| `current_order_count`             | 近1月共债订单数                          |
| `totaldebt_detail[].new_or_old=Y` | **疑似借新还旧**，任意月份命中即触发预警 |
| `totaldebt_detail` 趋势           | 共债机构数逐月递增，说明资金压力持续恶化 |

### Step 5：综合风险定级

| 风险等级          | 触发条件                                              | 准入建议                                 |
| ----------------- | ----------------------------------------------------- | ---------------------------------------- |
| 🔴 **高风险**     | 存在未结清逾期（`endFlag=N`）或 `new_or_old=Y` 命中   | **拒绝**                                 |
| 🟠 **中高风险**   | 逾期机构数 ≥ 3，或共债机构数 ≥ 5，或逾期总金额较大    | **人工复核**                             |
| 🟡 **中风险**     | 仅有已结清历史逾期（`endFlag=Y`），时间较近（≤6个月） | **审慎准入，降额**                       |
| 🟢 **低风险**     | 历史逾期均已结清且时间较远（>12个月），共债正常       | **正常准入**                             |
| ⚪ **数据未命中** | `code=1`，无记录                                      | 建议结合其他产品判断，不单独作为通过依据 |

## 输出模板

- **章节 1：风险结论 · Decision Pack**（风险等级 + 准入建议 + 核心命中信号）
- **章节 2：逾期维度分析**（近6个月逾期明细，含账期类型、金额、是否结清）
- **章节 3：共债维度分析**（近6个自然月共债趋势，借新还旧标识）
- **章节 4：综合评估与处置建议**（结合两维度给出授信建议或拒绝理由）
- **章节 5：数据来源与查询时间戳**

## 参数

- `--idNo`：身份证号（必填），末位 X 须大写
- `--idName`：姓名（必填）
- `--phoneNo`：手机号（可选，提升命中率）
- `--format md|json`：输出格式，默认 md

## 边界与免责

本 SKILL 基于三方支付数据，覆盖范围为有支付行为数据的用户群体，无法覆盖银行信贷体系内的正式征信记录。

`code=1`（未命中）不等于"无风险"，可能是该用户从未在三方支付体系内有借贷行为，建议配合央行征信或其他数据产品综合评估。

本 SKILL 输出为风险辅助参考，不构成最终信贷决策依据，授信决策须符合机构内部风控政策和监管要求。
