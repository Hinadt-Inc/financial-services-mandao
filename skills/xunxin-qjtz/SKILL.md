# 讯信-信用探查指数 · QJTZ 1.4.0

## SKILL 定位

本 SKILL 是讯信产品中**唯一具备正负双向信用评估能力**的探查类产品。相比履约指数（FMLH）只能判断"有逾期/无数据"两态，信用探查指数增加了两个关键维度：

- **正常履约识别**：`result_code=2`（B-Normal）直接标识申请人有良好的正常还款记录，是信用加分项
- **延迟履约区分**：`result_code=3`（B-Delay）识别轻度延迟还款行为，区别于严重逾期，支持精细化风险定价

同时返回履约正向字段（`max_performance_amt`、`count_performance` 等），可作为授信额度评估的正向参考。适用于希望同时做风险排除和优质客户识别的贷前审批场景。

**与履约指数（FMLH）的核心差异**：
- FMLH：仅返回 `result_code=1`（逾期）或 `4`（无数据），两态判断
- QJTZ：返回 `1/2/3/4` 四态，额外支持 B(Normal)/B(Delay) 识别，并返回履约正向字段

## MCP 依赖

- 必选：`mandao-company`（`getXunxinQjtzV140`）

## 通用执行原则

**第一，`result_code` 是核心判断入口，四态必须分别处理。** `1`（逾期）和 `2`（正常履约）是截然相反的结论，不能套用同一处置逻辑。

**第二，B(Delay) 不等于 A(Overdue)，需差异化对待。** `result_code=3` 表示延迟履约（轻度，通常为逾期1-15天内），属于边界风险，不应与严重逾期（M1+以上）等同处理，可通过降额、加强监测等方式准入。

**第三，`result_code=2` 是积极信号，应纳入授信加分项。** 正常履约说明申请人在三方支付体系内有良好还款记录，`max_performance_amt`（最大履约金额）可作为授信额度评估的参考上限。

**第四，`result_code=4`（U，无数据）需结合其他产品。** 无数据可能是新用户或线下信贷用户，建议配合综合指数V2（ZX-RadarV2）申请雷达维度补充判断。

**第五，`currently_overdue` 为实时当前逾期状态，权重最高。** 即便 `result_code=3`（延迟），若 `currently_overdue ≥ 2`，说明当前同时在多家机构逾期，实际风险等级需上调。

## 工作流

### Step 1：调用 MCP 获取数据

```
mcp__mandao-company__getXunxinQjtzV140
参数：{ "idNo": "<身份证号>", "idName": "<姓名>", "phoneNo": "<手机号>" }
```

> 底层支持 MD5 / SHA256 / SM3 三种加密，MCP 内部自动处理；身份证末位 X 须大写。

### Step 2：判断业务返回码

| data.code | 处理方式 |
|-----------|---------|
| `0` | 查询命中，进入 Step 3 解读 `result_detail` |
| `1` | 未命中，`result_detail=null`，输出"数据未命中"结论 |
| `9` | 系统异常，提示重新查询，不作风险结论 |

### Step 3：result_code 四态定性

| result_code | 含义 | 初始风险定性 |
|-------------|------|------------|
| `1` | **A(Overdue)**：有逾期记录 | 🔴 高风险，进入逾期详情分析 |
| `2` | **B(Normal)**：正常履约 | 🟢 正向信号，进入履约正向分析 |
| `3` | **B(Delay)**：延迟履约（轻度） | 🟡 边界风险，综合其他字段判断 |
| `4` | **U**：无数据/未知 | ⚪ 建议配合其他产品判断 |

### Step 4A：逾期维度解读（result_code=1 或 3 时执行）

| 字段 | 风险解读 |
|------|---------|
| `max_overdue_amt` | 最大逾期金额区间，金额越大风险越高 |
| `max_overdue_days` | 最长逾期天数：1-15天（轻微）→ 16-30天（中等）→ 31-60天（较重）→ >90天（严重） |
| `latest_overdue_time` | 最近逾期时间，越近权重越高（≤6个月为近期逾期） |
| `currently_overdue` | **当前逾期机构数**，≥2 为高风险，≥4 为极高风险 |
| `acc_exc` | 异常还款机构数（代扣失败/退款等），>0 提示资金流动异常 |

### Step 4B：履约正向维度解读（result_code=2 或 3 时执行）

| 字段 | 使用方式 |
|------|---------|
| `max_performance_amt` | 最大单笔履约金额，可作为本次授信参考上限 |
| `count_performance` | 历史履约笔数，笔数越多信用记录越丰富 |
| `latest_performance_time` | 最近履约时间，越近说明用户越活跃 |
| `currently_performance` | 当前履约机构数，反映正常还款中的借贷规模 |
| `acc_sleep` | 睡眠机构数，长期无行为的账户，影响有限 |

### Step 5：综合风险定级

| 风险等级 | 触发条件 | 准入建议 |
|---------|---------|---------|
| 🔴 **高风险** | `result_code=1` + `currently_overdue ≥ 2`，或 `max_overdue_days ≥ 31天` + 近期逾期 | **拒绝** |
| 🟠 **中高风险** | `result_code=1` + 历史逾期已结清，或 `result_code=3` + `currently_overdue ≥ 2` | **人工复核** |
| 🟡 **中风险** | `result_code=3`（延迟履约）+ `currently_overdue=0`，逾期天数 ≤ 15天 | **审慎准入，降额处理** |
| 🟢 **低风险** | `result_code=2`（正常履约），`currently_overdue=0` | **正常准入，参考履约金额上限授信** |
| ⚪ **数据未命中** | `code=1` 或 `result_code=4` | 建议配合综合指数V2或全景指数补充判断 |

## 输出模板

- **章节 1：信用状态结论 · Decision Pack**（result_code 四态定性 + 风险等级 + 准入建议）
- **章节 2：逾期维度分析**（逾期金额/天数/时间/当前逾期机构数，仅 result_code=1/3 时输出）
- **章节 3：履约正向维度分析**（履约金额/笔数/时间/当前履约机构数，仅 result_code=2/3 时输出）
- **章节 4：综合评估与授信建议**（正负信号交叉验证，输出授信建议或拒绝理由）
- **章节 5：与 FMLH 的差异说明**（如用户同时查询了履约指数，说明本产品额外返回的信号）
- **章节 6：数据来源与查询时间戳**

## 参数

- `--idNo`：身份证号（必填），末位 X 须大写
- `--idName`：姓名（必填）
- `--phoneNo`：手机号（可选，提升命中率）
- `--format md|json`：输出格式，默认 md

## 边界与免责

信用探查指数覆盖三方支付数据体系内的借贷行为，`B(Normal)` 正向信号仅反映该数据体系内的履约状态，不等同于央行征信体系的信用良好。

`result_code=4`（无数据）用户可能是优质的线下或银行体系用户，不应直接判定为高风险。

延迟履约（B-Delay）的具体阈值定义依讯信模型版本可能调整，建议以接口文档最新版本为准。
